# ユニットテスト基準

このファイルがユニットテストルールの Single Source of Truth。
TDD の原則は `tdd.md` を参照、より詳しい背景は `.kiro/steering/testing.md` を参照。

## テストを書く基準

### Tier 1: 必ずテストを書く

- **pure 関数 / pure モジュール** (`apps/local_input.py` / `apps/stitcher.py` 等)
  - 副作用を持たないので入力 → 出力の関係をテストすれば十分
  - 実装コストが低くリターンが大きい
- **オーケストレータの「呼び出し順序 / 引数 / 不変条件」**
  - 例: `local_auto_converter.process_pending` が Drive と Photos の **両方** にアップロードすること
  - 例: 失敗時に `mark_done` を呼ばないこと
- **ファイル命名規約 / フラグファイル規約のロジック**
  - Insta360 仕様の `_00_` / `_10_` ペアリング、`.auto_processing` / `.done` マーカー操作
  - 仕様変更時に回帰検知の役に立つ
- **境界値・エッジケース**
  - ペアの片目だけ存在 / done マーカーありで全部スキップ / アルバムが空

### Tier 2: 強く推奨

- **バグが出たことがある箇所** (`fix:` コミットを含む変更時)
  - 修正対象の関数にテストがあるか確認し、なければ追加
- **長期運用で副作用が見えづらい箇所**
  - `apps/utils.py` のログディレクトリ初期化、SMTP 周り
- **環境差異で挙動が変わる箇所**
  - 環境変数 (`INSTA360_LOGS_DIR` 等) で分岐する処理

### テスト不要

- **vendored ツール自体のテスト** (`Image-ExifTool-12.10/` / `spatial-media/` など)
  - 第三者ツールの再テストは責務外
- **Insta360 MediaSDK バイナリ自体のテスト**
  - NDA + 決定論性ゼロ
- **`refersh_gphotos_cred.py` のような OAuth 一発スクリプト**
  - ローカル実行する手動運用ツール

## モック / fake / 実 I/O の使い分け

| 対象 | 推奨 |
|---|---|
| Google API クライアント (Drive / Photos) | `unittest.mock.MagicMock` |
| `subprocess.Popen` / `subprocess.call` | MagicMock または `monkeypatch.setattr` |
| Insta360 MediaSDK | テストでは絶対呼ばない (thin wrapper をモック) |
| ファイル I/O | `tmp_path` fixture (実ディスクに書く) |
| 環境変数 | `monkeypatch.setenv` / `monkeypatch.delenv` |

**MagicMock のみで構築されたテスト** は実装と乖離しがちなので、
できるだけ `tmp_path` で実 I/O を回す方を選ぶ。

## カバレッジ方針

- 行カバレッジの目標値は **設定しない** (テストが少ない期間は無意味)
- ただし **新規追加・変更したコードパスは必ずテストで通す** ことを PR レビューで確認する
- 既存コードは TDD レガシー扱い: 触る箇所だけテストを足す

## テスト品質チェックリスト

- [ ] Tier 1 に該当するロジックにテストがあること
- [ ] Tier 2 に該当するロジックにテストの検討がされていること
- [ ] テスト不要な箇所に不必要なテストが追加されていないこと
- [ ] 正常系 + 異常系の両方が含まれていること
- [ ] 失敗シナリオで状態がロールバックされる (= マーカー作らない等) ことを検証しているか
- [ ] `tmp_path` で実 I/O を回す機会を見落としていないか? (MagicMock 偏重を避ける)
- [ ] 変更対象を使う既存テストの回帰リスクが確認されているか

## 迷ったときの判断基準

- このバグでユーザーに謝罪することになるか → Yes ならテスト必須
- 手動で確認するとき何パターン試す必要があるか → 3 パターン以上ならテストで自動化
- この関数を別の人が安心してリファクタできるか → 不安ならテストがセーフティネット
